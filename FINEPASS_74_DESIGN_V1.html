The 7.4 Fine Pass

Design V1 \u00b7 June 11 2026 \u00b7 differentiated numbers within the amateur cap \u00b7 V5-hybrid consistent

0 \u00b7 The problem, precisely

The amateur cap (no active amateur above STAR 7.4) is doctrine and stays. But the code rule appears to clamp rather than compress: every elite freshman whose raw evidence score exceeds the cap lands on exactly 7.4. Result: Dybantsa = Boozer = Peterson = Brown = Ament = Wilson = 7.4 \u2014 a pancake. The model's raw evaluation almost certainly DID differentiate them; the clamp throws that ordering away. (Same disease as the known R4 9.3 pancake at the top of the pro ladder.)

1 \u00b7 Design principle

Preserve the cap, preserve the ordering. The fix is code-side (a deterministic post-synthesis rule, per V5 hybrid doctrine) \u2014 never a prompt change with named-entity anchors (permanent LLM lesson: anchor lists amplify variance). The model keeps scoring raw and uncapped; the code maps raw into the band without flattening.

2 \u00b7 Two options

Option A \u00b7 Piecewise compression (deterministic, cohort-free)

Raw \u2264 7.4 \u2192 unchanged. Raw \u2208 (7.4, 9.4] \u2192 capped = 7.2 + 0.2 \u00d7 (raw \u2212 7.4) / 2.0 \u2192 lands in (7.2, 7.4]. One-decimal display yields 7.2 / 7.3 / 7.4 \u2014 three distinct values. Ordering preserved, no cohort dependence, fully reproducible from a single athlete's raw score. Limitation: granularity \u2014 six athletes share at most three display values.

Option B \u00b7 Two-decimal composites for capped amateurs

Same compression, but display two decimals (7.27, 7.34, 7.38\u2026) for athletes whose raw exceeded the cap \u2014 visually signals "elite amateur, capped, finely ranked." Limitation: product-surface change (tier ladder, PATH math, share card, draft board all read composite); inconsistent precision across the index needs design care.

Recommendation

Option A now, Option B evaluated after the draft. A is a one-rule code change, zero prompt risk, immediate de-pancake (7.2/7.3/7.4 beats six 7.4s), and the full 110-battery regression validates nothing else moved. B is a product decision Sridhar makes deliberately, not a calibration patch.

3 \u00b7 What it unlocks

  • Draft board sharpens: sealed values stay 7.4 forever (ledger immutability) \u2014 the board gains a "current grade" column next to the sealed one. "Sealed at 7.4, currently 7.3" demonstrates living index + frozen ledger in one row. That contrast IS the product story.
  • Movement events fire honestly: the fine pass itself is a release re-run \u2192 real composite-move events \u2192 first genuine entries on the wire and the verdict banners (FCBase85/88 light up with true data).
  • Pipeline view credibility: coach pool shows 6.7/6.4/6.1-style differentiation everywhere \u2014 the thing that made the prototype feel alive.
