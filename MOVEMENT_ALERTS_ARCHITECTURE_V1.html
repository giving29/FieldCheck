Movement Alerts \u00b7 Architecture V1

Internal design doc \u00b7 Phase 5 completion \u00b7 June 11 2026 \u00b7 "your number moved" engine

0 \u00b7 The product law (non-negotiable)

No fake movement, ever. An alert fires only when a real re-evaluation produced a real delta from a real event. One synthetic "your number moved" notification destroys the entire accountability brand. Silence beats noise.

1 \u00b7 What counts as movement

  • Tier jump \u2014 composite crosses a tier boundary in either direction (the headline event; FCBase53 alert UI already exists).
  • Facet delta \u2014 any facet moves \u2265 0.2 on re-evaluation ("Mental IQ 5.9 \u2192 6.2 after state semifinal").
  • Path delta \u2014 gap-to-target-rung shrinks/grows ("you are now 0.1 from STAR" \u2014 derived, costs nothing extra).
  • Coverage event \u2014 athlete enters the index for the first time ("your first verdict exists" \u2014 the acquisition alert).

2 \u00b7 Event sources (what triggers re-evaluation)

S1 \u00b7 Release re-runs (live today)

Every FCBase release re-runs the 110 battery + curated sets. Deltas between release N-1 and N are real movement. Zero new infrastructure \u2014 this is the V1 trigger.

S2 \u00b7 Scheduled refresh (cron exists)

The worker already has cron triggers (*/30 * * * *, 0 9 * * 1). Weekly: re-evaluate the watchlisted + marquee cohort (bounded cost), diff, emit events.

S3 \u00b7 Game-data ingestion (Ph7, later)

Post-game stat deltas via multi-source cascade (Tenet 53) trigger targeted re-evals. Design now, build after Friday \u2014 this is where "after every game" becomes literal.

3 \u00b7 Pipeline (all on existing stack \u2014 Worker + KV, no new vendors)

re-eval \u2192 diff vs KV snapshot \u2192 movement event \u2192 KV append \u2192 surfaces read

  • Snapshot: mv:snap:{slug} \u2192 {composite, tier, facets[8], release, ts}. Written on every evaluation.
  • Events: mv:events:{slug} \u2192 capped list (last 20) of {type, from, to, delta, reason, release, ts}.
  • Feed: mv:feed:global \u2192 last 100 events across the index (powers a public "Movement" wire \u2014 the dopamine page).
  • Diff rule: only S1/S2 evaluations write events; ad-hoc page views never do (idempotent, no view-triggered noise).

4 \u00b7 Surfaces (ship order \u2014 in-product first, push later)

M1
Verdict banner. "\u25b2 Moved since last release: Mental IQ +0.3" strip atop THE PATH, from mv:events. Pure read, additive patch to my own Path block.
M2
Watchlist badges. Existing watchlist v2 rows get \u25b2/\u25bc deltas \u2014 the reason to keep a watchlist.
M3
/movement wire. Public global feed page \u2014 "who moved this week" \u2014 newsroom-style. Shareable, SEO, recurring visit driver.
M4
Email/push opt-in. "Alert me when my number moves" \u2014 the retention contract. Needs provider choice (Netlify Forms \u2192 Resend, or Web Push). Post-Friday decision.

5 \u00b7 Honest-data guardrails

  • Derived (seeded) facets never emit facet-delta events \u2014 only curated/evaluated athletes do; derived athletes are eligible for tier + coverage events only (composite is real).
  • Every event carries reason + release \u2014 auditable on /agent-transparency principles.
  • Negative movement publishes identically. The ledger does not flinch.
