Always. Never skip. Promotion is Sridhar's decision on Sridhar's clock. Claude never proactively suggests ./deploy-prod.sh.
Rule 03
JS-validate after every patch
Syntax check before deploy. node --check or equivalent on every modified script.
Rule 04
Patch guards
Before applying a patch, check if it's already applied. Idempotent or skip.
Rule 05
Audit before assuming
Read the actual current state of the file/route/endpoint. Don't assume memory matches reality.
Rule 06
Surgical str_replace
Never rewrite a whole file when one block needs to change. str_replace the minimum, leave the rest untouched.

Update the canonical doc / hub BEFORE patching code. Strategy → tactics, never tactics chasing strategy.

Tenets 11 – 18
// Codified lessons · added per session · binding
"Each tenet was paid for in burned sprints. Re-read before every session."
— Operating principle
Tenet 11 · Downloads hygiene
Clean ~/Downloads after every cp
After every sprint, delete session HTML/patch files from ~/Downloads after cp into ~/Desktop/fieldcheck-proxy/. Keep baseline tarballs (FCBase*_*.tar.gz) — never delete. Prevents wrong-version drift.
Tenet 12 · Pre-deploy size check
ls -la + grep before deploy-dev
Before deploy-dev.sh: ls -la to confirm size matches expected sprint output. Grep for a known sprint feature. Mismatch = STOP, re-download. cp can succeed silently with the wrong source file.
Tenet 15 · Audit before patching
Grep selectors & function names
Verify CSS classes and function names against actual render code BEFORE patching. Before targeting .foo, grep class="foo". Before patching barFn(), grep its call sites. Codified after 4 burned sprints on .tl-stage (never existed) and buildMiniRadar (actual fn is biggerMiniRadar).
Tenet 15.1 · Satellite-class sweep NEW
List ALL classes, not just the wrapper
When removing a themed block, list every class used by that block's HTML and grep each. Ticker used .ticker AND .ti / .ti b / .ti .g / .ti .m / .ti .c. Searching only "ticker" missed all of them on first pass — caught on retry. Always grep: wrapper class · child classes · utility classes · keyframes referenced.
Tenet 16 · Doc rebuild after freeze
Update 3 MDs + rebuild HTML hub
Update all 3 markdown files (RELEASE_PLAYBOOK, ROADMAP_V3, GAME_PLAN) AND rebuild fieldcheck-roadmap.html after every baseline freeze. Announce per-doc changes in chat so Sridhar knows when to review the live URL.
Tenet 17 · Prototype before integrate
Standalone HTML for UX changes
For UX/CX changes — especially anything with animation, layout, or visual flow — build a standalone prototype HTML FIRST. Sridhar previews, picks, then integrate into the real page. Avoids deploying changes that don't visually match intent. FC16.S1–S5 cycle would have been 1–2 sprints instead of 5 with this rule.
Tenet 18 · Click-freeze diagnosis hierarchy NEW
Don't chase a freeze in the wrong file
Before debugging a click-freeze in the calling file, check the element for a JS handler. Plain anchor with no onclick = freeze lives downstream. Diagnose in this order: (1) handler exists → debug here · (2) plain anchor → check _redirects for that route · (3) route resolves to a file → debug that file's init · (4) SPA fallback (/* /index.html 200) → re-running homepage init may be the freeze itself.
Tenet · HTML-first output
Sridhar reads HTML, not MD
All roadmap / strategy / progress / changes updates go into existing HTML files — fieldcheck-roadmap.html for canonical hub, PLAYER_THESIS_V1_6_HUB.html for player thesis. Do NOT create new MD docs for content that belongs in existing HTML. Always present file after HTML update.
Tenet · Player Thesis Moat
Continuous learning loop
Moat = everyone trusting our player score. PhD team + case studies + draft outcomes + contract signings + on-field events form an infinite continuous learning loop refining the algorithm daily. Refinement is never "complete" — it IS the engine. Runs as parallel track alongside execution work. Never push Sridhar out of this loop.
Tenet · Jobs Doctrine
Einstein inside · Apple outside
Two layers. INSIDE = Einstein-grade depth (verdict moat, polygon, 16 voices — never simplify substance). OUTSIDE = Apple-simple UX (a 10-year-old could use it untaught). Four GOLD principles: (1) one mental model · (2) no dead ends · (3) implementation details invisible until needed · (4) self-healing state. Test: could a 10yo use this? If no, strip a layer.
