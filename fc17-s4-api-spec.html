v1.7 Posterior Service

Three endpoints. One worker route fork. Phased data layer: hardcoded → KV → D1. Designed to run parallel to FC17.S2A/B/S3 front-end work so the moment Phase 3 ships globally, the front end can retire hardcoded data and start fetching live.

Overview 3 endpoints

The new polygon needs structured v1.7 data — Bayesian posterior, refinement metadata, comparable trajectories. Three endpoints serve this; the existing fieldcheck-proxy.sridhar-nallani.workers.dev worker gets a new route fork at /api/v17/*. All other existing routes untouched.

Drop-in handler · add to existing worker.js
// ── FC17.S4 / Phase 4.7 · v17 API additions ──────────────
// Add this block to existing fieldcheck-proxy worker.js
// Goes BEFORE the existing catch-all fetch(request) at the end

// Phase 1: hardcoded data. See FLAGG_V17 in fc17-s2b-flagg-data.js
import { FLAGG_V17 } from './fc17-s2b-flagg-data.js';

const PLAYERS = {
  'cooper_flagg': FLAGG_V17,
  // add stephen_curry, madisen_skinner, etc as data prepared
};

function json(body, status = 200, extra = {}) {
  return new Response(JSON.stringify(body), {
    status,
    headers: {
      'Content-Type': 'application/json',
      'Cache-Control': 'public, max-age=300',
      'Access-Control-Allow-Origin': '*',
      'X-Framework-Version': '1.7',
      ...extra
    }
  });
}

function handleV17(request, url) {
  const path = url.pathname;
  const match = path.match(/^\/api\/v17\/(posterior|refinements|comparables)\/([a-z_]+)$/);
  if (!match) return json({ error: 'invalid_v17_route' }, 404);

  const [, resource, player] = match;
  const data = PLAYERS[player];
  if (!data) return json({ error: 'player_not_found', player }, 404);

  if (resource === 'posterior') return json(data.posterior);
  if (resource === 'refinements') return json(data.refinements,
    200, { 'Cache-Control': 'public, max-age=600' });
  if (resource === 'comparables') return json(data.comparables,
    200, { 'Cache-Control': 'public, max-age=3600' });
}

// In the main fetch handler, add early:
if (url.pathname.startsWith('/api/v17/')) {
  return handleV17(request, url);
}
// ...existing routes follow unchanged
Phase 2 migration path · When KV is ready, swap the PLAYERS object lookup for await env.V17_KV.get(`posterior:${player}`). The handler signature doesn't change — front end keeps fetching the same URLs.
Frontend integration pattern
Verdict page fetch · parallel + progressive render
// In fieldcheck-verdict.html · replaces hardcoded FLAGG_V17 once Phase 4.7 ships

async function loadV17(playerSlug) {
  const base = 'https://fieldcheck-proxy.sridhar-nallani.workers.dev/api/v17';
  const [posterior, refinements, comparables] = await Promise.all([
    fetch(`${base}/posterior/${playerSlug}`).then(r => r.json()),
    fetch(`${base}/refinements/${playerSlug}`).then(r => r.json()),
    fetch(`${base}/comparables/${playerSlug}`).then(r => r.json())
  ]);
  return { posterior, refinements, comparables };
}

// Player-gated rendering: new polygon if v17 data available, else old polygon
const v17 = await loadV17(playerSlug).catch(() => null);
if (v17 && v17.posterior && !v17.posterior.error) {
  renderV17Polygon(v17);
} else {
  renderLegacyPolygon();
}
Cross-cutting concerns
ConcernDecision
CORSAccess-Control-Allow-Origin: * · same-origin not required · public read-only data
AuthNone for reads (public verdicts). When Pro tier ships → bearer token for premium endpoints (separate spec).
Rate limitsCloudflare default per-IP limits. Cache hits don't count. If abuse → add per-IP quota in worker.
VersioningURL path versioned (/api/v17/). Future v1.8 = /api/v18/. X-Framework-Version header echoes for clients.
Cache strategyPosterior 5 min · refinements 10 min · comparables 1 hr (changes least). Bust by KV write timestamp (Phase 2+).
ObservabilityCloudflare Worker analytics for hit rate + latency. Custom metric: v17_endpoint_calls counter per endpoint.
Error budget99.5% uptime for read endpoints (well within Cloudflare SLA). Posterior service (Phase 3) has separate SLA.
Schema evolutionAdditive only within v17. Breaking change = new path (e.g. /api/v17.1/ or jump to /api/v18/).
