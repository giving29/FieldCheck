What the agent is

Claude Haiku 4.5 (Anthropic), called via a Cloudflare Worker. Each visitor question goes through one of 5 mode-specific system prompts depending on which page they ask from. The agent has access to only the athlete or page context we pass it — no browsing, no live data, no memory of other visitors.

Guardrails that apply across every mode

  • Hard cap on output: 1024 max tokens per answer (prevents runaway cost; ~2-3 paragraphs max).
  • Hard cap on input: 500 chars per question. Anything longer is rejected with HTTP 400.
  • Rate limit: 10 questions per IP per day. The 11th returns "rate limit exceeded" inline. Resets at midnight UTC.
  • No browsing. No live data. No memory across sessions or across visitors.
  • localStorage history only — last 3 Q&As per athlete on the visitor's own device. We never see it.
  • No advice. The agent will refuse recruiting decisions, draft predictions in a betting sense, NIL valuations, contract speculation, and personal-life questions.
  • Refuses out-of-scope. Methodology-page agent will not rate specific athletes. Predictions-page agent will not modify the ledger. Comparison-page agent will not pick a "winner" when composite gaps are under 0.3.
  • Sources are cited honestly. When the agent has thin data, it says so. A fully-sourced PROSPECT is treated as more reliable than a thinly-sourced STAR.

How to audit

If you ever think the agent is misbehaving, here's what to do.

  • Open your browser dev tools on any page hosting the agent (e.g. /methodology).
  • Tap the gold pill to open the chat panel.
  • Network tab → filter on /agent. You'll see the POST request with the exact mode + payload sent.
  • Console tab → check for any errors that came back as SSE events.
  • If the agent claimed something we don't grade, compare against the prompt above. The agent is constrained by these words — if the answer drifted, that's a real bug we want to know about.
  • Stats endpoint: /agent/stats returns aggregate usage counts (no PII).
