Data policy

Data policy.

Last updated: June 9 2026

FieldCheck operates on publicly available data. This page describes how we source it, how we process it, how often we refresh it, and how we handle conflicts.

Data sources

FieldCheck cascades through many sources. See the full source list and tier breakdown on the coverage map.

Tier 1 (primary): official school athletic sites, 247sports, On3, MaxPreps, ESPN, Sports Reference.

Tier 2 (cross-validation): CFBD, NBA/WNBA/NHL/MLB APIs, official athletic federation pages.

Tier 3 (context): Wikipedia, public news + press, curated PhD priors.

What we never use

  • Paywalled content we can't replicate.
  • Anonymous source quotes that can't be triangulated.
  • Private DM or off-record conversations.
  • Hacked / leaked private data.
  • Social media virality as a signal.

Refresh cadence

  • Photos · daily cascade refresh via worker proxy; 30-day edge cache, immutable.
  • Roster + recruit profiles · daily worker fetch.
  • Stats · real-time during in-season; daily off-season.
  • Verdicts · monthly PhD panel round for marquees; quarterly for full cohort.
  • Calibration weights · quarterly recalibration; weight updates published on the changelog.

Conflict resolution

When sources disagree (e.g. 247 ranks #1, On3 ranks #3), we cite both, prefer the higher-tier source, and surface the disagreement on the verdict's source-citation block. Disagreements aren't hidden; they're surfaced. Same logic applies to stats discrepancies, height/weight differences, etc.

Multi-source first (Tenet 53)

For any data type — photos, stats, bio, news, NIL — default to many sources with cascade. Never one. Wikipedia is one of many, not the default. Single-source dependency is a known failure mode; the multi-source cascade is the architectural defense.

If we got it wrong

Submit a correction via any verdict page's contact form. The dispute is reviewed by the PhD panel; if warranted, a versioned re-grade is issued publicly. We don't silently revise. We don't pretend the original verdict didn't exist. Versioned re-grades with diff notes are the standard.

Data we make available

Every verdict surfaces:

  • The 8-facet eval grid with per-facet scores
  • Source citations on every claim
  • Tier rationale
  • Confidence band on the composite
  • Calibration version that produced the grade

We don't paywall any of this. If you can read the verdict, you can read the sources.
