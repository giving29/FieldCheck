## BLOCK 10 — CRAWLER ARCHITECTURE SKETCH

**The biggest IP unlock per the canonical doc.** Here's the spec for review.

### The job
Find athletes the consensus missed by crawling YouTube + Hudl + Vimeo for game film, scoring the mechanics from the metadata, and surfacing the high-signal ones to `/hidden-gems`.

### Pipeline
```
1. CRAWL    YouTube Data API v3 → query: "{sport} highlights 2025 2026"
            Filter: views < 50,000  ·  upload_date < 90 days  ·  duration > 60s
            Output: raw video records (id, title, channel, description, views)

2. CLASSIFY For each record, extract player name + sport context from title/desc
            Uses Claude (haiku model for cost) → returns {player, sport, school, position}
            Drop records where confidence < 0.7

3. SCORE    For each classified record, run existing /api/video-iq pipeline
            (already built, sport-aware extraction is the asset we're reusing)
            Returns: composite + 11-dim signal scores

4. GATE     If composite > 7.5 AND player not in curated DB → discovery candidate
            Else: drop (too low signal) OR upgrade (already-known player → update verdict)

5. HOLD     Discovery candidate → write to KV with status="pending_review"
            Notify Sridhar via Slack webhook (or email)

6. PUBLISH  Sridhar reviews + approves → status="published"
            Discovery surfaces on /hidden-gems "Discovered by Video IQ" section
            (frontend ✅ already there)
```

### Cost
- YouTube Data API v3: 10,000 quota units/day free → ~100 search calls/day → batch into 10 calls × 100 results = 1,000 videos/day reviewed
- Claude Haiku classify: ~$0.25/1M input tokens → ~$0.50/day for 1,000 classifications
- /api/video-iq runs: free (existing infra)
- **Total: < $20/month for 30K video reviews/month**

### Worker contract (what I'll spec when greenlit)
```js
// new worker route
async function handleCrawl(env, request) {
  // 1. Pull seed queries from KV (per-sport, rotated)
  // 2. Hit YouTube API, get fresh records since last_crawl_at
  // 3. Pass each to classifyVideo() → {player, sport, confidence}
  // 4. Pass classified → existing scoreFromVideoMetadata()
  // 5. Gate → KV write to "discoveries:pending:{id}"
  // 6. Webhook notify
  return new Response(JSON.stringify({reviewed, qualified}));
}
// Trigger: cron every 6h via Cloudflare Workers Cron Triggers
```

### Risks
- YouTube rate limits → mitigated by quota budget + batch
- Classification false positives → human review gate catches
- Sridhar review queue overflow → cap candidates at top-20 per cron run

**Status:** Ready to spec for worker deploy when you greenlight.

## BLOCK 11 — MODEL INTEGRATION ON /ADD-ME SKETCH

### Flow
```
Athlete lands on /add-me
  ↓
Pastes YouTube/Hudl URL + 5 self-fields (position, school, year, stats, why-good)
  ↓
Backend: same /api/video-iq pipeline + their self-assessment as a source layer
  ↓
Return: side-by-side
  [Self-assessment polygon] | [Model polygon]
  Divergences flagged (where athlete overrates / underrates themselves)
  ↓
Athlete reviews + opts in to "publish to Hidden Gems if score ≥ 8.0"
  ↓
If opt-in + qualified → enters the discovery queue
  ↓
Athlete gets a shareable verdict card (URL) regardless
```

### Why this is moat-grade
- **Athlete-vs-model divergence** is its own data signal. Athletes who overrate by a lot signal one type; underraters another.
- **Training set bootstrap.** Each /add-me submission with athlete-supplied stats is labeled data for Phase 5B crawler.
- **The first FC-grade evaluation an unranked kid has ever gotten.** Moat from the demand side.
