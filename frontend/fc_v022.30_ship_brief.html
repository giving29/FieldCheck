Each athlete gets 6 specific litmus checks. The aggregate determines GREEN / YELLOW / RED.

6brutal_honest_presentPatch F worked · encyclopedia.brutal_honest.ok === true with confidence + ≥1 evidence_anchor

How GREEN / YELLOW / RED is determined

GREEN = all checks pass (no fail, no near, no null). YELLOW = no checks failed but some are "near" the expected range (within 0.5 of band edge) or current_school is null but expected band allows it. RED = at least one check failed outright. SKIP = no response received or HTTP error (dev was down or timed out).

SECTION 06

4-persona walkthrough framework

Once batch test hits ≥90% green, pick 5 athletes (one per tier) and manually verdict-page them. Each persona walks through with a stopwatch. 90-second win condition per persona.
★ PLAYER · 30-SECOND BAR
Where do I stack? What's next?
Test athlete: a real HS prospect opens the page about themselves. In 30 seconds they must see: (1) Where do I stack vs others at my tier — concrete percentile, not vague praise; (2) What's my trajectory — improving / steady / declining with evidence; (3) Realistic path to my goal school — named programs, not vague tiers; (4) The 1-2 things I can do in the next 6 months. Win: Player texts a friend the verdict and the friend understands it without context.
★ COACH · 30-SECOND BAR
Recruit, pass, or watch?
Test athlete: a college coach evaluating for roster slot. In 30 seconds: (1) Fit for our scheme; (2) Slot projection (starter / rotation / depth / red-shirt); (3) Recruiting status; (4) Red flags (academics, transfer history, character); (5) The honest "ask". Win: Coach forwards to assistant with "Pull tape" or "Pass" in under a minute.
★ SCOUT · 60-SECOND BAR
Project: ceiling / floor / comparable?
Test athlete: pro/independent scout building a draft board. In 60 seconds: (1) Comparable archetype with named pros; (2) Ceiling/floor/median outcome with confidence; (3) Hidden indicators others would miss; (4) Scout-grid letter grade per facet; (5) Where we differ from public composite + why. Win: Scout can defend this read in a scouting meeting against pushback.
★ PARENT · 90-SECOND BAR
Should we commit? What's the real cost-benefit?
Test athlete: parent of HS junior weighing D1 vs D2 vs D3 vs walk-on. In 90 seconds: (1) Realistic placement — not inflated; (2) Academic + financial reality; (3) Decision framework — the 3 questions to ask; (4) Time horizon; (5) The honest read — what would FieldCheck tell a friend about this kid. Win: Parent shares with spouse and they jointly decide without needing a recruiting consultant.

The Patch F brutally honest section is the persona-level winner

Patch F gives every verdict page encyclopedia.brutal_honest.decision_grade_read with one sentence each for for_player / for_coach / for_scout / for_parent. That's the test. If those four sentences are honest and useful, the page passes the 90-second bar. If they're vague platitudes, Patch F prompt needs refinement.

SECTION 07

Tenets 41-44 · codify so this never regresses

After V022.30 ships, add these to FC_CANONICAL_STATE_V1.html V5.22. They prevent future Claudes from sliding back into the spaghetti that caused the Antwan Kimmons cascade.
TENET 41 · IDENTITY VALIDATION AT EVERY ADAPTER BOUNDARY
Any adapter that resolves to a player-specific profile MUST validate identity match before returning ok=true
Wikipedia, Basketball-Reference, PFR, Baseball-Ref, 247, On3, MaxPreps, Hudl — each must validate the profile-page name matches the queried name (first-name initial + last-name whole-word + Lev≤1 on last). Fuzzy-matching "first /players/ link" without validation is a SEVERE bug. Codified after V022.30 (May 26 2026): Antwan Kimmons query fuzzy-matched a Spurs HOFer's page → bbref_pro returned ok=true with is_hof=true → composite forced to 9.7 via pathway_floor → entire verdict corrupted across all 5 amateur tests. Single-adapter failure cascaded through 4 layers downstream.
TENET 42 · MULTI-SOURCE CORROBORATION FOR HIGH-IMPACT CLAIMS
Any signal that triggers a CASCADE requires ≥2 independent sources OR ≥1 source + structured accolade evidence
Career-stage promotion (especially legend_pro), composite floor overrides, tier classification — none may fire from a single noisy adapter. Threshold: hofSourceCount ≥ 2 OR (hofSourceCount ≥ 1 AND structuredAccolades > 0). Single-source claims trigger the cascade only when the source is structured-data-validated (Wikidata Q-ID, school roster confirmed, NCAA Stats Portal direct match).
TENET 43 · NO DESTRUCTIVE OVERWRITES
Pathway floors and tier adjustments preserve the original signal AND require explicit confidence ≥ threshold to overwrite display value
When pathway floor fires (e.g. legend_pro pushes 3.5 → 9.7), the original score MUST be preserved as composite_pre_pathway_floor AND the display composite shows the original 3.5 with the floor as a possibility annotation IF confidence is below threshold. Only high-confidence overrides display the floored value as truth. Brutal honesty over inflated tier promotion.
TENET 44 · END-TO-END VISUAL VERIFICATION BEFORE "SHIP"
Worker JSON correct is necessary but not sufficient. Ship-ready requires browser-rendered page check + ≥3 amateur tiers + ≥1 pro regression check + persona walkthrough
A successful curl JSON does not equal a working product. Required for ship: (1) curl JSON has all 8 facets populated + identity.current_school correct + correct tier in position_pool; (2) verdict page renders correctly in browser at dev URL; (3) at least 3 amateur athletes (HS + D1 + D2 OR D3) all pass; (4) at least 1 historical pro (Cooper Flagg or similar) regression check confirms no break for upper tier; (5) 4-persona walkthrough passes 90-second win condition on 5 representative athletes. Failing ANY of (1)-(5) BLOCKS the prod promote.
