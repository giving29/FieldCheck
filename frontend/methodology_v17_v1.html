01Why we built this

Sports-tech evaluators have a structural integrity problem. The buyer is usually a team paying for a coach decision, an agent paying for a contract argument, or a media outlet paying for a daily-content cycle. None of those buyers pay for honest "we don't know yet." So evaluators hedge, smooth, and quietly delete bad calls.

FieldCheck's v1.7 was built around a different bet: that retail evaluators — parents, scouts, analysts, fans, individual coaches — will pay for verdict integrity if we publish the wrong ones. The framework below is what makes that promise auditable rather than rhetorical.

The shortcut to understanding v1.7 is this: every number on a FieldCheck verdict has a credible interval, a sign-off panel, and a check-in date. If a number ages badly, you'll find it on the prediction ledger. If a number ages well, you'll find the same audit trail. The framework is the same either way.

02The 8 facets

What changes is the display label per sport (a volleyball player's "Captain DNA" maps to the same underlying canonical pillar as a basketball player's "Mindset"). The underlying scoring stays comparable across sports.

Each facet returns a posterior distribution — not a point estimate. We publish the mean, the 95% credible interval (CI), and a confidence score that captures how much evidence informs the posterior. Wide CIs mean we admit uncertainty. Tight CIs mean we have strong evidence and have committed.

03Why Character uses MIN, not average

Most evaluator frameworks average the components of "Character." A player rated honest 9, reliable 9, but harm-avoiding 6 would average to 8 and get a green light. That averaging hides the failure mode: character risks cascade. A high-honesty, high-reliability player who lacks harm-avoidance can still produce a catastrophic event.

v1.7's Character pillar applies the Yamamoto MIN function: the score equals the minimum of the 5 sub-dimensions, not the average. If any single sub-dim drops below the threshold, the pillar drops with it.

For Cooper Flagg, this means Character returns 8.5 (harm-avoidance · untested adult-stress) instead of the 9.06 average. The MIN-vs-average gap is the whole point. We surface the limiting factor instead of hiding it.

Categorical events (DUIs, character incidents) are never blended into the score arithmetically. They're displayed separately as documented events. If an event activates the R6 cascade refinement (Character below 7), the entire composite gets a -1.5 penalty applied transparently — not absorbed into the smoothing.

04The 6 refinements

A v1.7 verdict can have up to six refinements applied. Each one is a published formula. Active refinements show up on the verdict polygon as small icons; inactive ones are documented but don't affect the score.

🩹
R1 · injury risk on Physical
Substrate × (1 − injury_risk)
When durability data exists, Physical's effective score discounts for projected injury risk. effective = substrate × (1 − injury_risk). Flagg: 9.5 × (1 − 0.04) ≈ 9.12, rounded with frame + recovery factor to 9.3.

⚖️

effective = mean × pressure_multiplier
Adjusts Mental Strength for pre-draft / pre-major-stage context. Not active for Flagg — post-draft, pressure inflection already cleared.
Inactive
🔄
R3 · Talent × Physical interaction
composite_lift = if (T ≠ P) sqrt(T × P) else mean(T, P)
When Talent and Physical diverge meaningfully (≥0.8 gap), an interaction term replaces the simple average. Not active for Flagg — Talent (9.7) and Physical (9.3) too close.
Inactive
🎯
R4 · coachability bias correction
0.4 × coach + 0.3 × teammates + 0.3 × observers
Coach quotes carry optimism bias. The weighted blend pulls toward observer + teammate signal. Flagg: 0.4×9.4 + 0.3×8.8 + 0.3×9.0 = 9.04, rounded to 9.0.
Active · Flagg
📍
R5 · CI widening on untested sub-dim
sd_eff = sd_base × (1 + 0.3 × untested_count)
When a sub-dimension is untested in a relevant stress context, we widen the credible interval rather than penalize the mean. Honest is not the same as bearish. Flagg: harm-avoidance untested in adult-stress → sd 0.35 → 0.45.
Active · Flagg
⚖️
R6 · cascade refinement (low Character)
if Character < 7 then composite − 1.5
If Character mean drops below the threshold (e.g., after a documented event), the composite takes a flat penalty. Transparent, not absorbed. Not active for Flagg — Character at 8.5, well above 7.
Inactive

Refinements are part of the framework lock. We don't add new ones between versions. R1-R6 are the v1.7 universe of corrections. If we find we need an R7, that means v1.8 — and v1.8 doesn't retroactively change v1.7's locked verdicts.

05The 25-PhD panel

A FieldCheck verdict isn't computed and pushed live. It's reviewed by a standing panel of 25 PhDs spanning sports science, psychometrics, biomechanics, behavioral economics, and statistics. Every locked verdict has the panel sign-off documented in the audit trail.

Dissents are part of the record. Cooper Flagg's verdict carries a documented dissent from Dr. Vance (panelist · psychometrics) who argued the Character CI should widen further. That dissent didn't block the lock, but it didn't disappear either. When the August 2028 check-in arrives, Vance's dissent gets the same scrutiny as the consensus call.

The panel doesn't operate in real time. Verdicts are reviewed in monthly batches. Between batches, the framework parameters (weights, thresholds, sub-dimension definitions) are immutable. This is the structural defense against drift, recency bias, and reactive smoothing.

06Pre-registration before outcomes

Every v1.7 verdict is pre-registered: locked at a specific timestamp with a specific check-in date, before the outcomes that would prove the verdict right or wrong are knowable. For Cooper Flagg, that's May 17, 2026 lock · August 15, 2028 check-in. We committed to the verdict before he played his second full NBA season.

Between lock and check-in, the verdict cannot be edited. A great rookie season doesn't move the composite up. An injury doesn't move it down. New evidence informs the next verdict; it does not retroactively change the one being graded.

This is the discipline that makes the framework auditable. If we let new information into a locked verdict, we'd be grading our forecasts against shifting goalposts. The whole point of pre-registration is to tie our own hands.

Outcomes get ingested into the corpus weekly. The PhD panel reviews calibration monthly. Pillar weights can shift quarterly based on out-of-sample performance. Refinements (R1-R6) only change at annual review. Major version changes (v1.8) happen on a five-year cycle. None of these cadences retroactively change locked verdicts.

08How we publish failures

Every verdict that closes "wrong" gets published on the public prediction ledger. By name. With the verdict we issued, the outcome that happened, and what we got wrong about the player. Three definitions of "wrong" in increasing severity:

// out-of-band wrong
Empirical composite outside CI
3-year measured composite falls outside the credible interval we published. If we said 9.30 ± 0.5 and reality returns 7.8, we were wrong, full stop.
// wrong-tier wrong
Tier delta ≥ 2 levels
Empirical tier is at least 2 tiers below predicted tier. ELITE+ predicted, SOLID-STARTER delivered.
// cascade wrong
R6 activation event
A categorical event drops Character below 7 (R6 cascade fires) or a major injury collapses Physical. Documented with whether the cascade was predicted as a watch-list trigger.
// partial · documented
Edge cases
Trajectory inside CI but tier transition different (e.g., HOF lock but no All-NBA appearances). Published as partial-right with delta narrative.

A verdict closes right only when the empirical 3-year composite falls inside the CI and the empirical tier matches or exceeds the predicted tier. Anything else gets graded down.

09The 6 constraints, unchanged

Behind the framework sit six structural constraints. They are not negotiable. They are the part of FieldCheck that doesn't move when business pressure tries to move it.

1
Free tier permanent
The Free tier exists forever. We never sunset it, never paywall the core verdict, never lock the polygon behind a sign-in.
2
No affiliate revenue
FieldCheck takes no money from agencies, no money from leagues, no money from advertisers paying to influence verdicts. The user pays. Nobody else.
3
No payola
No team, agent, or player buys their way to a higher score. If we ever take such payment, every prior verdict is invalidated retroactively.
4
Walk-away default
If a verdict requires compromising the framework (e.g., a high-paying client demands their player be graded differently), we walk away. Loud, in public, with the reason published.
5
Failures stay published
Wrong verdicts go on the ledger. They stay on the ledger. We don't delete them when v1.8 ships. The historical record of failure is part of the credibility.
6
Framework lock
Locked verdicts cannot be edited between framework versions. We grade our own work against the framework we used at lock-time, not the framework we evolved to.

10The V5 algorithm · prompt + deterministic post-processing

V022.36-V5.2 · MAY 27, 2026 · BATTERY VALIDATED 110/110

The V5 algorithm is a two-layer hybrid system: a prompt-driven synthesis layer evaluates evidence, and a deterministic post-synthesis correction layer applies bright-line rules where LLM variance is unacceptable. Audit trail in composite_v022_31.v5_corrections[] shows every adjustment with rule, raw value, corrected value, and reason.

10.1 · Layer 1 · Synthesis (prompt-driven)

Sonnet 4.5 evaluates the athlete against 8 facets with cross-sport theoretical 10 anchors, 4 phenom criteria, 8 anti-inflation signals, and the default-low principle. Synthesis output includes raw composite and per-facet scores. 10 is the only ceiling. No mathematical caps applied at this layer — Sonnet scores based purely on evidence rigor.

Prompt sections that drive the rigor:

  • Cross-sport theoretical 10 anchors — Duncan / Brady / Tiger / Federer per facet calibrate upper bounds across sport contexts
  • Phenom criteria — 4 reqs locked: national-best evidence, pro coach corroboration, multi-year sustained signal, cross-context consistency. All 4 must be true for phenom_qualified.
  • Anti-inflation — 8 explicit signals that do NOT justify higher scores (recruiting rank, summer camp, hype articles, etc.)
  • Default-low principle — when evidence is uncertain, score lower; V5 inverts V4's hype-default-up
  • Anti-contamination — reasoning instructions for Jr/Sr/family-lineage cases with named examples (Kate Harpring → score the daughter, not Matt Harpring NBA father)
  • Critical 9.7+ reserved — imperative upper-bound rules: NO active player exceeds 9.6 except retired-singular Jordan at 9.9

What was removed: The V5 v2.1 iteration tried adding HARD NUMERIC ANCHORS — a list of named athletes with specific score ranges. This backfired. Saniyah Hall was anchored at 5.0-5.4 with the list present, yet Sonnet returned her at 9.0. The reason: listing Saniyah alongside Caitlin Clark, Steph Curry, and Tim Duncan caused peer-association bias — the model associated her with their tier, overriding the numeric bound. This is a generalizable LLM lesson: reasoning instructions ("if X, then Y") beat anchor lookup tables for deterministic behavior. We removed the list and moved enforcement to Layer 2.

10.2 · Layer 2 · Deterministic post-synthesis correction

After synthesis returns raw composite, a JavaScript function _v5_apply_corrections applies 6 bright-line rules. Each rule is conditional on identity, career stage, and tier signals — derived from the design doc, not arbitrary tier numbers.

Rule Trigger Action
R1Jr/Sr/II/III/IV suffix (regex) + amateur tiercomposite ≤ 5.5 (HS) or 7.4 (D1)
R2HS + prep_amateur + no phenom flagcomposite ≤ 5.4
R3D1 + college_amateur + no phenom flagcomposite ≤ 7.4
R4Active career stage (rookie/early/prime/late_pro)composite ≤ 9.3
R5Rookie career stagecomposite ≤ 7.5
R6HS tier + pro stage (classification mismatch)flag identity-review-needed (no correction)

Why this is not V4 caps. V4 had unconditional tier-based ceilings with no transparency. V5 corrections are conditional on identity+stage+tier signals; every correction logs rule, from, to, and a reason in the audit trail; raw synthesis is preserved in metadata; and phenom-qualified athletes bypass R2 and R3 entirely.

10.4 · Validation · 110-athlete battery

The architecture validated against 110 athletes across 11 buckets (60 HS / 17 D1 / 13 D2 / 10 D3 / 3 JUCO / 7 pro regression+collision). Cold cache against DEV endpoint. Results:

  • 100% rule compliance on the amateur moat case (HS + D1 + D2 + D3)
  • Jr/Sr suffix athletes deterministically bounded — Jordan Smith Jr 9.83 → 5.4, Carlos Medlock Jr 9.0 → 5.4, Eric Booth Jr 7.5 → 5.4
  • Top D1 amateurs at evidence ceiling — Cameron Boozer 9.1 → 7.4, AJ Dybantsa 9.3 → 7.4, JT Toppin 8.7 → 7.4
  • Active pros at active ceiling — Cooper Flagg 9.71 → 9.3 (active rule), Tim Duncan retired passes through unchanged at 9.3
  • Under-cap variance preserved — Kate Harpring 4.0, Manuella Fernandez 4.0, Jamarion Vincent 5.3 (D1 hidden gem), Anthony Brown Jr 6.2

10.5 · Design lessons from 3 iterations

  • V5 v1 rolled back — caps off, evidence rigor weak. Every HS scored 9.6-9.8. Pure prompt iteration without rigor is naive.
  • V5 v2 — added cross-sport anchors, phenom criteria, anti-inflation. Clean cases worked. Contamination cases inflated.
  • V5 v2.1 — added HARD NUMERIC ANCHORS list. Backfired: Saniyah Hall 3.5 → 9.0 via peer-association. Named-entity lists amplify variance.
  • V5 v2.2 + v2.3 — rolled back the list, added deterministic post-processor + audit. 100% compliance. Hybrid prompt+code is the architecture.

Generalizable insight: for any LLM-scored system that requires bright-line behavior, hybrid wins. Use prompts for evidence reasoning (LLM strength). Use code for identity/pattern detection (deterministic). Preserve raw output in audit metadata for transparency.
