1. The thesis · why we built this

Every scout grade is a black box. 247Sports gives you "92" with no math. ESPN gives you "5 stars" with no math. Hudl shows tape with no synthesis. MaxPreps shows stats with no context. Recruiting industry: 100% opaque.

FieldCheck V5 is built around the opposite principle: show the composite, show every adjustment that produced it, show the reason for each. Parents see how their kid's score was reached. Coaches see what evidence drove it. Scouts see the methodology. Athletes see the path to the next tier.

The moat is transparency

We don't beat scout grades by being more accurate (LLM variance is real). We don't beat them by having more data (yet — Block 10 crawler is Phase 6). We beat them by being the only score in the entire sport that shows its work. Raw synthesis, every correction, every reason, every flag.

You can't claim transparency if you don't publish the algorithm. So here it is.

2. Architecture · two layers + audit trail

The V5 algorithm is a hybrid system. Layer 1 is prompt-driven; Layer 2 is deterministic code. They serve different purposes.

LayerRoleWhy this design
Layer 1 · synthesisSonnet 4.5 reads athlete evidence, scores 8 facets, produces raw compositeLLMs are excellent at evidence evaluation, cross-context reasoning, phenom assessment
Layer 2 · correctionsJavaScript function applies 6 bright-line rules with audit loggingPattern detection (Jr suffix regex), conditional logic, and consistency need determinism — LLMs vary run-to-run
Audit trailEvery correction logged in v5_corrections[] with rule, from, to, reasonTransparency moat — every other scoring system is opaque

3. Layer 1 · synthesis (evidence eval)

Sonnet 4.5 evaluates the athlete against 8 facets with cross-sport theoretical 10 anchors, 4 phenom criteria, 8 anti-inflation signals, and default-low principle.

10 is the only ceiling. No mathematical caps at this layer — Sonnet scores based purely on evidence rigor. Cross-sport anchors (Duncan, Brady, Tiger, Federer) calibrate the upper bound of each facet across sport contexts.

Prompt sections (the rigor)

  • CROSS-SPORT THEORETICAL 10 ANCHORS — Duncan, Brady, Tiger, Federer per facet — calibrate facet-level upper bounds
  • PHENOM CRITERIA — 4 reqs locked: national-best evidence + pro coach corroboration + multi-year sustained + cross-context performance
  • ANTI-INFLATION — 8 explicit signals that do NOT justify higher scores (recruiting rank, summer camp, hype articles, etc.)
  • DEFAULT-LOW PRINCIPLE — when evidence is uncertain, score lower; V5 inverts V4's hype-default-up
  • ANTI-CONTAMINATION — reasoning instructions for Jr/Sr/family-lineage cases with named examples
  • CRITICAL 9.7+ RESERVED — imperative upper-bound rules ("NO active player exceeds 9.6 except retired-singular Jordan at 9.9")

What was removed

In V5 v2.1 we tried adding a HARD NUMERIC ANCHORS list — named athletes with specific score ranges. This backfired catastrophically. Saniyah Hall was anchored at 5.0-5.4. With the list present, Sonnet returned her at 9.0. Why?

The peer-association bias. Listing Saniyah Hall alongside Caitlin Clark, Steph Curry, and Tim Duncan in a single anchor table caused Sonnet to associate her with their tier — overriding the numeric bound right next to her name. Lists of named entities in prompts amplify variance.

This is a generalizable LLM lesson: reasoning instructions ("if X, then Y") beat anchor lookup tables for deterministic behavior. We removed the list and moved enforcement to Layer 2.

4. Layer 2 · deterministic corrections (6 rules)

After synthesis returns raw composite, _v5_apply_corrections applies 6 bright-line rules. Each is conditional on identity, career stage, and tier — derived from the design doc, not arbitrary tier numbers.

RuleTriggerActionWhy this rule
R1 Jr/Sr/II/III/IV suffix (regex) + amateur tier composite ≤ 5.5 (HS) or 7.4 (D1) Parent's pro career data contaminates synthesis context. Score the amateur, never inherit the parent.
R2 HS + prep_amateur + no phenom flag composite ≤ 5.4 HS-only evidence cannot justify above 5.4 unless all 4 phenom criteria are met.
R3 D1 + college_amateur + no phenom flag composite ≤ 7.4 D1 amateur sample is single-year for most. Phenom exception preserved.
R4 Active career stage (rookie/early/prime/late_pro) composite ≤ 9.3 9.7+ reserved for Jordan retired singular. Active careers cap at multi-MVP band 9.3.
R5 Rookie career stage composite ≤ 7.5 Rookie sample is limited regardless of draft position or recruiting hype.
R6 HS tier + pro stage flag identity-review-needed (no auto-correction) Logical mismatch suggests name collision. Flag for manual review; don't silently correct.

The code

const _v5_apply_corrections = (rawComposite, identity, careerStage, tier, phenomFlag) => {
  let corrected = rawComposite;
  const corrections = [];
  const name = (identity && (identity.full_name || identity.name)) || '';
  const suffixRegex = /\b(Jr\.?|Sr\.?|II|III|IV|2nd|3rd)\b\s*$/i;
  const hasSuffix = suffixRegex.test(name.trim());
  const activeStages = ['rookie', 'early_pro', 'prime_pro', 'late_pro'];

  // R1 · Jr/Sr suffix + amateur → parent contamination
  if (hasSuffix && (careerStage === 'prep_amateur' || careerStage === 'college_amateur')) {
    const ceiling = careerStage === 'prep_amateur' ? 5.5 : 7.4;
    if (corrected > ceiling) {
      corrections.push({ rule: 'suffix-amateur-ceiling', from: corrected, to: ceiling,
        reason: 'Jr/Sr/II/III suffix at amateur tier - parent career data likely contaminated synthesis' });
      corrected = ceiling;
    }
  }

  // R2 · HS evidence ceiling (non-phenom)
  if (tier === 'hs' && careerStage === 'prep_amateur' && !phenomFlag && corrected > 5.4) {
    corrections.push({ rule: 'hs-evidence-ceiling-non-phenom', from: corrected, to: 5.4,
      reason: 'HS-only evidence, no phenom criteria met (4 of 4 required)' });
    corrected = 5.4;
  }

  // R3 · D1 amateur ceiling (non-phenom)
  if (tier === 'd1' && careerStage === 'college_amateur' && !phenomFlag && corrected > 7.4) {
    corrections.push({ rule: 'd1-college-amateur-ceiling', from: corrected, to: 7.4,
      reason: 'D1 amateur evidence ceiling' });
    corrected = 7.4;
  }

  // R4 · active player ceiling (Jordan retired singular at 9.9)
  if (activeStages.includes(careerStage) && corrected > 9.6) {
    corrections.push({ rule: 'active-player-ceiling', from: corrected, to: 9.3,
      reason: '9.7+ reserved for Jordan retired singular; active multi-MVP cap 9.3' });
    corrected = 9.3;
  }

  // R5 · rookie ceiling
  if (careerStage === 'rookie' && corrected > 7.5) {
    corrections.push({ rule: 'rookie-evidence-ceiling', from: corrected, to: 7.5,
      reason: 'Rookie sample - draft slot/recruiting rank does not elevate' });
    corrected = 7.5;
  }

  // R6 · HS + pro stage = classification mismatch flag
  if (tier === 'hs' && activeStages.includes(careerStage)) {
    corrections.push({ rule: 'identity-classification-mismatch', flag: 'review-needed',
      reason: 'HS tier classified with pro stage - likely name collision' });
  }

  return { composite: corrected, raw_synthesis: rawComposite, corrections };
};

5. The audit trail · transparency on every score

For every athlete, the verdict page can render the full audit chain from raw synthesis to final composite. This is what users see when they click "Why this score?"

The transparency promise

If a correction was applied, we show:

  • The raw synthesis value (what Sonnet 4.5 produced from evidence)
  • Each rule that fired, in order
  • The from-value and to-value for each correction
  • The reason in plain language ("Jr suffix detected — parent's pro career likely in synthesis context")
  • The final composite

If no correction was applied, we show: "Synthesis output passed through cleanly — no bright-line rules fired."

This is the moat. Every parent, coach, scout, and athlete can see the methodology applied to their specific case. No black box.

7. What we learned · 3 iterations

  1. V5 v1 (rolled back) — caps off, evidence rigor weak. Every HS scored 9.6-9.8. Lesson: pure prompt iteration without evidence rigor is naive.
  2. V5 v2 — added cross-sport anchors, phenom criteria, anti-inflation. Clean cases worked. Contamination cases (Jr suffix, family lineage) inflated. Lesson: prompt reasoning handles general cases; deterministic patterns need code.
  3. V5 v2.1 — added HARD NUMERIC ANCHORS list with named athletes. Backfired: Saniyah Hall 3.5 → 9.0 from peer-association bias. Lesson: named-entity lists in prompts amplify variance.
  4. V5 v2.2 + v2.3 — rolled back the anchor list, added deterministic post-processor with 6 bright-line rules + audit trail. 100% compliance. Lesson: hybrid prompt+code is the architecture.
Generalizable insight: For any LLM-scored system requiring bright-line behavior, hybrid wins. Use prompts for evidence reasoning (LLM strength). Use code for identity/pattern detection (deterministic). Preserve raw output in audit metadata for transparency.

8. Open items · what's next

Acknowledged limitations of V022.36-V5.2

  • Synthesis None errors (~13 athletes) — Sonnet doesn't complete on some inputs (timeout or parse failure). Patterns: D2/D3 athletes with thin evidence, some HS. Phase D1 data layer fix.
  • Identity classification instability — Career stage detector has ~5% run-to-run variance (Caitlin Clark hs↔d1, Antwan Kimmons hs↔pro). Rules fire correctly given whatever synthesis returns, but inputs to rules could be cleaner. Phase C1 validation gate next sprint.
  • Phenom criteria not firing — Currently no athlete passes phenom_qualified=true from synthesis. Means R2/R3 cap Boozer/Dybantsa/Flagg even if they're legitimate phenoms. Next sprint: add phenom_qualified output field with strict 4-criteria.
