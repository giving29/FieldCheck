The API · Embed widgets · Reuse data · No key required

Embed it. Reuse it.

FieldCheck-LOVB is built to be reused. Every widget, every dataset, every surface is available for embedding into broadcaster pre-game packages, journalism pieces, club pages, scout databases. No API key. No rate limit. CC BY-SA 4.0 license.

§ I · Quick start

Three minutes to embed.

Drop a single script tag and a div placeholder. The widget renders with FieldCheck identity. All data refreshes when our backend re-syncs from LOVB sources.

Step 1. Pick the widget you want. The full menu is in §III below; atlas is a good default.

Step 2. Paste this into the page where the widget should appear:

<!-- FieldCheck-LOVB Atlas widget --> <div id="fc-lovb-atlas"></div> <script src="https://fieldcheck-app.netlify.app/embed/atlas.js"></script>

Step 3. Verify it renders. The widget loads inside the div with our identity (Anton + Instrument Serif + Crimson Pro). Inherits the host page's background; otherwise self-contained.

That's the whole flow. No API key, no auth, no rate limiting.

§ II · Data layer access

The raw data file.

If you want full data without the widget UI — pulling for your own analysis or building your own visualizations — just fetch the static JSON-able data file. Same source of truth, same updates.

// Fetch the data file fetch("https://fieldcheck-app.netlify.app/fc-lovb-data.js") .then(r => r.text()) .then(js => { // The file assigns window.LOVB (and aliases LOVB_DATA, FC_LOVB) const script = document.createElement("script"); script.text = js; document.head.appendChild(script); console.log(window.LOVB.markets); });

The exported window.LOVB contains: markets (6 LOVB Pro teams + 3 expansion), foundingAthletes (10 athletes), icons2026 (14 athletes), juniorClubs (90 affiliated), statLeaders2026, standings_2026, foundation, schedule_2026, and helper functions under LOVB.helpers.

Every record is sourced — each has a source field pointing to the LOVB page or named publication that grounds the claim. If you publish from our data, please cite the original source we cite, not just us.

§ IV · Endpoints reference

Read-only HTTP endpoints.

If you want to fetch specific data slices instead of the full data file, these endpoints return JSON. All read-only, all GET, no auth.

GET
/api/markets
All 6 Pro markets + 3 expansion markets. Includes ownership, standings, founders, junior affiliates.
GET
/api/markets/:code
Single market by code (atl/atx/htx/mad/neb/sl). Full dossier data.
GET
/api/athletes
All athletes — Founding Ten, 2026 Icons, full LOVB Pro rosters when available. Filterable by team, position, status.
GET
/api/icons/2026
2026 LOVB Icons First Team + Second Team. Unanimous selections flagged. Source-linked.
GET
/api/standings
Current season standings. Updates on each data sync (logged at /lovb/changelog).
GET
/api/leaders/:category
Stat leaders by category (points/kills/blocks/assists/digs). Top six per category.
GET
/api/clubs
All 90 affiliated junior clubs. Filterable by state, by Pro affiliate, by 2025 national-champion status.
GET
/api/schedule/2026
Full 2026 LOVB Pro regular-season + playoffs schedule. Broadcast partner attribution included.

§ VI · Versioning + stability

What won't break on you.

If you're embedding widgets in broadcaster pre-game packages or scout databases, you need to know what's stable. Here's what we commit to.

Widget script URLs are stable. The script at /embed/atlas.js will resolve to the same widget identity for at least 24 months. If we deprecate a widget, we'll provide migration notes 6 months ahead in /lovb/changelog.

Data shape is stable. The window.LOVB object keys won't be renamed or removed without a version bump. We add fields freely; we don't remove or rename them.

Endpoint paths are stable. Same 24-month commitment as widget URLs. New endpoints come; existing ones don't change shape without notice.

Sync cadence is best-effort. Our backend re-syncs from LOVB sources approximately weekly. During playoffs and major news (Icons announcement, ownership announcements), we sync ad-hoc. Stale-data risk is low; if you see a clearly outdated number, file a dispute.
