// SECURITY · EDGE TIER · FOR ATHLETIC DEPTS · NIL COLLECTIVES · PRO CLUBS

Audited intelligence. Auditable infrastructure.

FieldCheck Edge runs the same engine as the public site — at higher trust commitments. This page documents how authentication, encryption, audit logging, and data processing work for institutional buyers.

// 01 · AUTHENTICATION

SAML 2.0 SSO + scoped bearer tokens

SSO via SAML 2.0 with major IdPs (Okta, Azure AD, Google Workspace). For API access, scoped bearer tokens with per-endpoint rate limits. Bearer tokens carry tier metadata so a Dept token can't hit Pro endpoints.

// 02 · ENCRYPTION

TLS 1.3 in transit · AES-256 at rest

All traffic TLS 1.3 end-to-end. All persisted data AES-256 at rest. Database backups encrypted with separate KMS keys. No customer data ever stored on local dev machines.

// 03 · AUDIT LOGS

Every Edge action logged · 90-day default

Every Edge user action — login, verdict request, dispute submission, export — is logged with timestamp, user, tier, payload hash. Retention 90 days default, 1 year on Institution+ tier. Exportable to your SIEM.

// 04 · DATA PROCESSING

Per-customer DPA on file

Data Processing Agreement signed before any data flows. Lists what we collect, why, how long, who processes it. Sub-processor list maintained publicly. Customer's own roster data never used to train shared models.

// 05 · SOURCE TRANSPARENCY

Every claim cites a public URL

The audit envelope (see /api) lists every public source URL for every claim in every verdict. There is no "proprietary scout database" to compromise — because there is no proprietary scout database. The integrity surface is public.

// 06 · INCIDENT RESPONSE

72-hour disclosure SLA

Any incident affecting customer data: notification within 72 hours. Post-mortem published within 14 business days. Pattern of unrelated incidents triggers Methodology Desk review per the Edge contract.

// 07 · METHODOLOGY INTEGRITY

SHA-256 every verdict

Every verdict signed. Reproducible — re-run the same input, get the same hash or a drift flag. The methodology lives in public at /methodology. Version diffs at /changelog.

// 08 · ROSTER PRIVACY

Customer watchlists never aggregated

The Coach Workspace v2 watchlist sits in your browser's localStorage by default — not on our servers. Even on Edge Institution+ tier where boards are server-side, customer watchlists are never aggregated into market intelligence across tenants.
